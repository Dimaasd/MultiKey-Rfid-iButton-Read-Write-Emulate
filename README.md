# MultiKey-Rfid-iButton-Read-Write-Emulate

Пристрій призначений для читання, копіювання, збереження та емуляції ключів домофонів різних типів.

Він підтримує як контактні ключі Dallas (iButton), так і безконтактні RFID-мітки стандарту EM-Marine 125 кгц

Початковий скетч - це робота [AlexMalov/EasyKeyDublicatorRFID_OLED](https://github.com/AlexMalov/EasyKeyDublicatorRFID_OLED) (т.н. проект "MEXATРОН DIY")
- змінені деякі елементи схеми і коду (запущена емуляція iButton)
- замість енкодера, три кнопки
- видалено виведення інформації в серійний порт (для економії місця в пам'яті в ардуїно)
- дисплей: OLED 128x32 (I2C) хотів поміняти на - 128x64 (I2C), але пам'ять ардуіно обмежена - бібліотеки не влазять
- котушка L1 345 мкГн - замінена на 736 мкГн (провід 0,25 мм, 155 витків, діаметр 20 мм, ширина 2 мм, корпус котушки.stl)
- конденсатор для резенансу С1 4.7нФ - замінений на 2,2нФ
  
Основна проблема добитись резонансу в контурі, 
потріно намотати на котушку більше витків (180 наприклад),
а потім в режимі зчитування (горить зелений діод) замірювати напругу на контрольних точках TP1 і TP2,
відмотуючи по одному витку з котушки контролюємо зміну напруги, 
при досягненні найвищої напруги - фіксуємо край проводу,
У мене зчитує. записує, емулює з 5-6 сантиметрів,
корпус котушки перед 3Д друком розрізав у слайсері на три частини- два борти і центр,
а потім склеював усе разом, борти з гладкою стороною зазташовував в середину, 
щоб при намотці провід не зачипався об зазубрини погрішностей друку,
Використав корпус павербанку, по назві можна знайти на аліекспресі або де інде, 
макетна плата помістилась замість двох акумуляторів 18650, зручно із живлення не подрібно додаткового вимикача,
одне натискання на бокову кнопку павербанку включає плату і заодно подається напруга 5 вольт на ардуіно нано,
сама плата павербанку не переходить в сон від низького зпоживання,
виключається подвійним натисканням на цю ж кнопку.
задня кришка закручується гвинтами.

🔧 Апаратна частина:

Мікроконтролер: Arduino Nano (ATmega328P),
- Дисплей: OLED 128x32 (I2C)
- Інтерфейс керування: 3 тактові кнопки
- Індикація: RGB-світлодіод + окремий світлодіод
- Звуковий супровід: п'єзодинамік
- Живлення: від USB або зовнішнього джерела

📌 Призначення виводів (Pinout):
- Пін	 - Призначення	   - Опис
- A3	 - iButtonPin	     - Лінія даних iButton
- 10   - iBtnEmulPin	   - Емуляція iButton
- 8	- R_Led	RGB - червоний
- 2	- G_Led	RGB -  зелений
- 12 - B_Led	RGB - синій
- 6	- ACpin	- Вхід аналогового компаратора
- 4	- speakerPin	- П'єзодинамік
- 11	- FreqGen	- Генератор 125 кГц для RFID
- A1	- BTN_MODE	- Кнопка режиму
- A2	- BTN_LEFT	- Кнопка "Лівий"
- A0	- BTN_RIGHT	- Кнопка "Правий"
  
📦 Пам'ять та зберігання:
- EEPROM: зберігає до 42 ключів з назвами

Кожен ключ зберігається як структура KeyData:
- 8 байт - ID ключа
- 16 байт - текстова назва

🗝️ Підтримувані типи ключів:
- Dallas (iButton) - контактні ключі
- Cyfral - контактні
- Metacom - контактні
- TM2004 - контактні
- EM-Marine - безконтактні (RFID 125 кГц)

🎮 Режими роботи (перемикання кнопкою MODE):
- Read (зелений) - режим читання ключів
- Write (червоний) - режим запису на чистий ключ
- BlueMode (синій) - режим емуляції ключа

🕹️ Керування:
- Дія	- Функція
- MODE (коротке натискання)	- Перемикання режимів (Read → Write → BlueMode)
- LEFT/RIGHT	- Перегортання збережених ключів
- MODE (утримання в Read) - 	Збереження поточного ключа в EEPROM
- MODE (утримання в Write) - 	Видалення поточного ключа (з підтвердженням)
- MODE (під час емуляції)	- Зупинка емуляції

🔄 Функціональні можливості:
- Читання ключів
- Автоматичне визначення типу ключа (Dallas, Cyfral, Metacom, EM-Marine) при піднесенні до зчитувача.

- Запис ключів:
- Підтримка різних типів чистих заготовок:
- TM2004
- RW1990-1 / RW1990-2
- TM01
- T5557 (для RFID)

- Емуляція ключів:
- Dallas: використовує бібліотеку OneWireHub для точної емуляції
- EM-Marine: генерація сигналу 125 кГц

- Керування пам'яттю:
- Збереження прочитаних ключів
- Видалення окремих ключів
- Перегляд списку ключів

- Автоматичне визначення дублікатів

🔊 Звукова індикація:
- Старт пристрою - мелодія
- Успішне читання/запис - звуковий сигнал
- Помилка - тривожний сигнал
- Початок/кінець емуляції - спеціальний сигнал
- Видалення ключа - низький звук

💡 Світлодіодна індикація:
- Зелений - режим читання
- Червоний - режим запису
- Синій - режим емуляції RFID-мітки
- Блимання синім - активна емуляція iButton

📟 Дисплей (OLED 128x32) Відображає:
- Поточний режим
- Номер ключа та загальну кількість
- HEX-код ключа (8 байт)
- Тип ключа
- Назву ключа (якщо є)
- Підказки для користувача

📚 Вбудовані універсальні ключі
- Пристрій містить набір універсальних ключів:
- Домофон
- Гараж
- UK-3 Cyfral
- Universal
- Vizit 99%
- Forward
- Cyfral
- Metakom 95%

🛠️ Використані бібліотеки:
- OneWire - робота з шиною 1-Wire
- OneWireSlave - емуляція пристроїв 1-Wire
- OneWireHub - розширена емуляція
- DS2401 - віртуальний пристрій Dallas
- GyverOLED - робота з OLED дисплеєм
- TimerOne - таймер для генерації сигналів
- pitches.h - ноти для звукових сигналів
- EEPROM - робота з внутрішньою пам'яттю

⚙️ Технічні деталі:
- Частота генератора RFID: 125 кГц
- Максимальна кількість ключів: 42
- Антидребезг кнопок: 50 мс
- Таймаут утримання: 1 секунда
- Автоматичне завершення емуляції: 30 секунд
